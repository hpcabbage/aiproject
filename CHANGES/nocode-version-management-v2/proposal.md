# Proposal - nocode-version-management-v2

## Why

当前 NoCode 项目已经完成版本管理第一阶段最小闭环：
- 手动提交版本
- 版本列表
- 版本详情
- 版本备注
- 回滚
- 真实文件版本目录 + 数据库索引

但当前体验仍然更接近“版本快照列表”，还不是一个面向 NoCode 用户可理解、可操作、可回溯的版本管理中心。

用户当前仍缺少这些关键感知：
- 哪一版正在被使用
- 哪一版是稳定版本
- 哪一版是回滚生成
- 某条版本记录与另一条版本记录之间是什么关系

因此需要把版本管理从 V1 的“数据可用”，升级为 V2 的“产品可理解”。

## What

本次 change 聚焦：

1. 为版本系统增加更强的产品语义表达
2. 把版本列表升级为版本管理中心而不只是数据列表
3. 明确当前版本、稳定版本、回滚来源版本等关系
4. 保持 NoCode 视角，不引入代码 diff

本次 change 预期覆盖：
- 数据模型升级
- 后端版本语义补强
- 前端版本列表 / 详情页升级
- 文档与规格同步

## Out of Scope

本次明确不做：
- 面向用户的代码 diff
- 自动推断“本次改了什么”
- 视觉 diff / 文件 diff
- 完整发布时间线高级组件
- 多人协作版本权限系统
- 发布中心 / 灰度 / 审核流
